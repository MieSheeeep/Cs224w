# GCN (Graph Convolutional Network) 核心机制与思考笔记

## 第一部分：GCNConv 是什么卷积方式？

GCNConv 不是图片那种网格卷积（固定 3×3 窗口扫像素），而是一种**基于图拓扑的聚合卷积**。
* **官方名称**：谱域图卷积（一阶近似）
* **通俗理解**：邻居聚合卷积。每个节点只聚合**【自己 + 直接邻居】**的特征。

### 1. GCNConv 如何计算？（核心 4 步）
GCN 对每个节点的核心操作：**新特征 = 归一化加权求和（自己 + 邻居特征）**。
具体步骤如下：
1.  **加自环**：节点自己也算自己的邻居。
2.  **归一化**：防止度数大的节点“声音太大”。
3.  **聚合**：把自己 + 所有邻居的特征加起来。
4.  **线性变换 + 激活**：乘以权重矩阵并激活，变成新特征。

### 2. 为什么能捕捉图的拓扑结构？
因为 GCN 让**节点特征 = 自己 + 邻居的信息**。邻居是谁、有多少、连接关系，全部融入了特征里！
* 例如：节点 1 连着 2、3；节点 4 连着 5、6。它们聚合出来的特征天然不一样，这就自动学到了图的拓扑结构。
* **感受野机制**：层数越深，感受野越大（1 层看 1 阶邻居，2 层看 2 阶邻居的邻居...）。

### 3. 手算例子：GCNConv 到底怎么算？
以最简单的小图为例：
* **图结构**：`1 — 2`，且 `1 — 3` (节点 1 连着 2 和 3)
    * 节点 1 的邻居：2、3（度为 2）
    * 节点 2、3 的邻居：1（度均为 1）
* **节点初始特征（1维）**：node1: `[1]`, node2: `[2]`, node3: `[3]`

**GCN 计算（以计算 node1 为例）：**
1.  **加自环**：自己也算邻居 → 聚合对象：1、2、3。
2.  **归一化权重**：公式简化为 $权重 = 1 / \sqrt{度(u) \times 度(v)}$
    * $w(1 \rightarrow 1) = 1 / \sqrt{2 \times 2} = 0.5$
    * $w(1 \rightarrow 2) = 1 / \sqrt{2 \times 1} \approx 0.707$
    * $w(1 \rightarrow 3) = 1 / \sqrt{2 \times 1} \approx 0.707$
3.  **加权求和（核心）**：
    * $新特征 = 0.5 \times 1 + 0.707 \times 2 + 0.707 \times 3 = 4.035$
    * 然后再乘上可学习的权重矩阵 $W$。

**结论**：node1 聚合后的特征（0.4035或相应变换后的值）融合了【自己的特征】+【邻居的特征】+【图的连接结构】。

### 4. GCNConv 终极总结
代码体现：`h = self.conv1(x, edge_index)` （`edge_index` 携带了图的拓扑结构）
* **GCNConv = 邻居聚合卷积**，不是网格卷积。
* **计算 = 归一化加权求和（自己 + 邻居）**。
* **能学拓扑 = 因为聚合天然依赖图的连接关系**。

## 第二部分：GCN 与半监督学习 (Semi-supervised)

**为什么 GCN 是半监督学习？**
用少量标签 + 大量无标签数据一起训练。
1.  图里大部分节点没有标签（比如 2708 个节点，只有 140 个有标签）。
2.  模型能利用整张图的结构，学习**所有**节点的特征。
3.  **训练时只使用有标签的节点算损失 (Loss)**。

**代码级理解：**
```python
loss = criterion(out[data.train_mask], data.y[data.train_mask])
```
* **全局预测**：`out` 包含了图里**每一个节点**的预测结果。
* **局部损失**：只挑出 `train_mask` 标记的那极少数有标签的节点来算误差、更新模型。
* **全局参与**：大量无标签节点不参与损失计算，但它们的**连接关系、特征、邻居**参与了特征学习。

## 第三部分：深度思考与灵魂拷问（你的真实思考轨迹）

> **🤔 你的思考与疑问 1：**
> “我的理解：也就是说它在前向传播的时候会直接计算所有图的数据点，然后我们得到一个用只有标签节点的 loss，但是我们通过这个loss去计算 gradient 并反向传播，这个反向传播会传播到每一个节点的嵌入向量？并以此做更新？所以我们的目的仅仅是预测该图内的数据点，而不能是其他图的。”

**核心解答：**

* **前向传播**：接收所有节点特征和拓扑结构，对**每个节点**进行特征聚合，输出所有预测结果。
* **损失计算**：只用带标签节点算 Loss。
* **反向传播**：更新的是模型参数（W）。参数更新后，**所有节点的嵌入向量会因为参数变化而被间接更新**。
* **适用范围**：
    * **半监督节点分类**：确实是预测**当前这张图**内的无标签节点，不能直接迁移到全新图（因为拓扑和分布不同）。
    * **图分类任务**：预测**整张图**（如分子毒性预测），这种模型学的是跨图的通用规律，可以迁移。
* **为什么少量 loss 能带动全图？** 1. 拓扑信息传播（标签信息沿着边传播给无标签节点）。
    2. 梯度全局关联（参数 $W$ 是全局共享的）。

> **🤔 你的思考与疑问 2：**
> “但是我观察模型结构，是否 GCNConv 只有最后的 W 是可学习的，对周围节点特征的初次权重计算是不可学习的。然后还有一个可学习的点在全连接层。然后既然 loss 会和无标签的点没有关系，那不是意味着它们嵌入层是无法被学习的，只有有标签的节点的嵌入才能被反向传播到？”

**核心解答（纠偏）：**
前两点观察完全正确（GCNConv 只有 $W$ 可学习，归一化公式固定；Linear 层可学习）。**但结论完全错误！**

**真相是：所有节点（有标签 + 无标签）的嵌入都会被学习、被更新！**

1.  **反向传播更新的不是“节点嵌入”，而是模型参数 $W$**。
2.  **$W$ 是全局共享的**，无标签节点和有标签节点用的是同一套 $W$。
3.  只要有标签节点的 loss 更新了 $W$，下一次前向传播时，**所有节点（包括无标签）用新的 $W$ 算出来的嵌入就全部自动更新了**。它们是被动跟着 $W$ 一起学习的。

> **🤔 你的思考与疑问 3：**
> “所以嵌入的本质是什么，似乎就是模型参数本身了？ 所以其实这个模型就是在训练嵌入？ 训练完成后，如何体现它的嵌入？是将所有点的初始特征输入，然后经过模型运算得到嵌入？ 所以初始特征一样的点嵌入一样了，核心在于它没有更新过特征，为什么它能够分类？”

**核心解答：**
* **嵌入的本质不是参数**：嵌入是“固定公式算出来的中间结果”。
$$\text{embedding}^{(l+1)} = \sigma(\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2} \cdot \text{embedding}^{(l)} \cdot W^{(l)})$$
    嵌入 = 拓扑结构 + 节点特征 + 模型参数 的结合产物。
* **如何体现嵌入**：训练完成后，将初始特征和拓扑结构输入，运行到 GCNConv 输出层，得到的就是最终嵌入（可以通过画图工具根据类别可视化这些点在空间中的聚集情况）。
* **初始特征一样的点，嵌入会一样吗？** **不一定！** 因为嵌入不仅依赖初始特征，还依赖**拓扑结构**。如果你和我的特征一样，但你的邻居是 A 类，我的邻居是 B 类，我们经过 GCN 聚合后的嵌入就会截然不同！这就是它能分类的核心依据：它带上了“结构身份”。

> “好了，那么又回到那个问题上，嵌入是什么，是模型在进入全连接层之前吗？”

**核心解答：完全正确！嵌入就是模型进入全连接层之前的中间向量。**

结合代码看一目了然，嵌入就是那个“桥梁” `h`：

```python
class GCN(nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels):
        super().__init__()
        # 1. GCN卷积层（提取图特征）
        self.conv1 = GCNConv(in_channels, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, hidden_channels)
        # 2. 全连接层（分类器，把嵌入转成类别得分）
        self.lin = nn.Linear(hidden_channels, out_channels)

    def forward(self, x, edge_index):
        # 第一步：第一层卷积 → 中间特征
        h = self.conv1(x, edge_index).relu()
        # 第二步：第二层卷积 → 【这就是最终的节点嵌入！】
        h = self.conv2(h, edge_index)  
        # 第三步：全连接层 → 【把嵌入转成类别预测】
        out = self.lin(h)  # 嵌入 h 作为全连接层的输入
        return out, h
```
